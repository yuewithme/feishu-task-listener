Feishu Automation Hub

一个围绕飞书事件订阅构建的自动化入口。它接收群聊工单消息，解析负责人和任务类型，写入多维表格， 私发可交互任务卡片，并在异常时把处理信息回送给机器人创建者。

Core Workflow

群工单主流程

页面重点不是列功能，而是让人一眼看懂消息从进入飞书到落表、发卡、回写状态的全过程。

01

群消息进入

机器人只处理明确 @机器人 的消息，普通聊天消息会被跳过。

  • im.message.receive_v1
  • 去重 message_id
  • 过滤未 @ bot 的消息
02

解析任务字段

从消息文本和 mention 结构里提取执行人、任务类型、发布时间和可选车牌号。

  • 执行人来自第一位 user mention
  • 任务类型基于关键词映射
  • 车牌号允许为空
03

写入多维表格

将结构化记录写入 Base，并保留原始消息文本，方便后续追溯和人工补录。

  • 任务发布时间
  • 执行人
  • 任务类型
  • 消息原文
04

私发任务卡片

向执行人私发卡片，包含领取任务、已解决、跳转记录补位置等交互入口。

  • 发送身份：bot
  • 目标：执行人 open_id
  • 卡片支持 update_multi
05

按钮回写状态

点击后回写领取时间或解决时间，并更新卡片按钮状态，防止重复点击。

  • claim → 处理中
  • resolve → 已解决
  • 追加填写位置的跳转按钮
06

异常处理

发送失败、字段缺失、用户授权失效时，统一给机器人创建者发送异常处理卡片。

  • 群聊名称
  • 消息原文
  • 失败原因
  • 目标执行人身份信息

关键文件

把入口、路由和各业务脚本拆开，便于定位责任边界。
